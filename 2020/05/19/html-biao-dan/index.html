HTML表单

HTML表单是用户和web站点或应用程序之间交互的主要内容之一。它们允许用户将数据发送到web站点。大多数情况下，数据被发送到web服务器，但是web页面也可以自己拦截它并使用它。

HTML表单是由一个或多个小部件组成的。这些小部件可以是文本字段(单行或多行)、选择框、按钮、复选框或单选按钮。大多数情况下，这些小部件与描述其目的的标签配对——正确实现的标签能够清楚地指示视力正常的用户和盲人用户输入表单所需的内容。

HTML表单和常规HTML文档的主要区别在于，大多数情况下，表单收集的数据被发送到web服务器。在这种情况下，您需要设置一个web服务器来接收和处理数据。如何设置这样的服务器超出了本文的范围，但是如果您想了解更多，请参阅模块后面的发送表单数据。

本篇中为了进行演示，可能直接将示例代码放在其代码块下面，让它能够插入到生成的html页面中。但这在经过博客主题的渲染后，有些插入的表单控件不能用了，而有些则出现了非常奇特的风格。

构建表单元素

form元素

此元素定义了一个表单域，其中放置表单小控件。

以下两个形式是禁止的：

  • 嵌套<form>元素；
  • 将表单控件放在<form>之外（尽管这是允许的，而且HTML5中提供了一些机制如form属性来支持，但其在跨浏览器的实现中还不够，所以还是不用了）。

所有HTML表单都以一个<form>元素开始：

这个元素正式定义了一个表单。就像<div>元素或<p>元素，它是一个容器元素，但它也支持一些特定的属性来配置表单的行为方式。它的所有属性都是可选的，但实践中最好至少要设置action属性和method属性。

  • action 属性定义了在提交表单时,应该把所收集的数据送给谁(/那个模块)(URL)去处理。.
  • method 属性定义了发送数据的HTTP方法(它可以是“get”或“post”)。

fieldset和legend元素

一个<form>元素内可能有许多的表单控件，出于语义、方便、样式管理的需要，还有fieldset标签来将相同目的的表单控件进行包装（可以看做是子表单域）。在其中，需要先添加一个<legend>文本元素，来说明这个子表单域的内容。

其最大的用处：屏幕阅读器可以去阅读它。

比如下面的例子

<form>
  <fieldset>
    <legend>Fruit juice size</legend>
    <p>
      <input type="radio" name="size" id="size_1" value="small">
      <label for="size_1">Small</label>
    </p>
    <p>
      <input type="radio" name="size" id="size_2" value="medium">
      <label for="size_2">Medium</label>
    </p>
    <p>
      <input type="radio" name="size" id="size_3" value="large">
      <label for="size_3">Large</label>
    </p>
  </fieldset>
</form>
Fruit juice size

label元素

label元素来定义表单控件的标签。既然其是为了为某个表单控件提供标签，所以其就有特殊的属性来实现和特定表单控件之间的联系，即for属性。

可以看到，label和legend实际上和span没什么区别，都是为了装文本用的。那为什么还要发明这么多的文本元素呢？这就得又重复一次HTML的宗旨：语义、语义、还是语义。发明这些元素是为了让每段话都有其自己的目的或语义，这使得我们的代码容易理解，更加重要的是，其对于一些又阅读障碍的人来说，是非常必要的。

如下面的例子：

<label for="name">Name:</label> <input type="text" id="name" name="user_name">

for属性中需要写其连接的表单控件元素的id（也就是说，其对应的表单控件必须写id）。这样屏幕阅读器可以读出label元素中的文本。

而另一个更加重要的、这样设置label元素的原因是：我们可以通过点击这个标签来激活相链接的表单控件元素。

使用更多其他的HTML元素

实际上，表单作为HTML的一部分，我们可以通过去利用其他的更多元素来美化它。使得它更加结构化、各个部分之间的划分更加清晰等等。这可能需要用到div、p、section、h1等诸多元素。

这里提供了一个指导：

  1. 可以使用一些文本元素h或p首先进行一些描述；
  2. 使用section来划分表单的不同的部分；
  3. 使用fieldset将一些同类型的表单控件包裹在一起，这些表单控件可以使用列表元素进行进一步的组织（radio、checkbox一般需要进行如此的操作）；
  4. 对其中存在的文本可以使用strong、abbr等增加其表现能力。

表单控件

这里介绍的是HTML自己自带的表单控件，当然也可以自己建立表单控件，这将在之后进行介绍。

以下是所有表单控件的通用属性：

  • autofocus=false，这个布尔属性允许您指定当页面加载时元素应该自动具有输入焦点，除非用户覆盖它，例如通过键入不同的控件。文档中只有一个与表单相关的元素可以指定这个属性。
  • disabled=false，这个布尔属性表示用户不能与元素交互。如果没有指定这个属性，元素将从包含它的元素继承设置，例如<fieldset>;如果没有包含在设定了disabled属性的元素里，那么这个元素就是可用的。
  • form，小部件与之相关联的表单元素。属性值必需是同个文档中的<form> 元素的 id属性。理论上，它允许您在<form>元素之外设置一个表单小部件。然而，在实践中，没有任何支持该特性的浏览器。
  • name，元素的名称;这是跟表单数据一起提交的。
  • value，元素的初始值。

1. 文本输入框

使用type="text"的input元素来实现，其通过使用不同的属性，可以完成不同的任务。

其有一些通用的规范：

  • 可以使用readonly或disable来禁止用户修改值或者甚至根本不发送值。
  • 使用placeholder来给文本框中输入一些值，这些值不是默认值，而是一些对文本框目的的解释性语言。
  • 可以使用size（框的物理大小）和maxlength（可以输入的最大字符数）来限制其大小。
  • 可以从拼写检查中获益。

1.1 单行文本框

基本单行文本框范式：

<input type="text" id="comment" name="comment" value="I'm a text field">

type可以给一些特殊的值，但其实际上还是单行文本框，只是添加了一些额外的约束和特性。

  1. Email地址框

    <input type="email" id="email" name="email" multiple>

    当输入的不是有效的电子邮件地址时，会报错。multiple属性允许输入多个地址，使用,分割。

  2. 密码框

    <input type="password" id="pwd" name="pwd">

    在输入文字的使用，使用点或星号来显示。但注意，在发送内容的时候其依然是使用明文发送的，所以最佳的保护方式是使用https进行加密。

  3. 搜索框

    <input type="search" id="search" name="search">

    主要是样式不同。

  4. 电子号码栏

    <input type="tel" id="tel" name="tel">

    只是语义上的需要。

  5. URL栏

    <input type="url" id="url" name="url">

    无效的url会报错。

通过list属性链接一个datalist元素，我们可以为输入框提供备选值，其示例如下：

<label for="myFruit">What's your favorite fruit?</label>
<input type="text" name="myFruit" id="myFruit" list="mySuggestion">
<datalist id="mySuggestion">
  <option>Apple</option>
  <option>Banana</option>
  <option>Blackberry</option>
  <option>Blueberry</option>
  <option>Lemon</option>
  <option>Lychee</option>
  <option>Peach</option>
  <option>Pear</option>
</datalist>

1.2 多行文本框

注意，多行文本框是个特例，其不用input，而是使用textarea。其主要区别在于，允许用户输入换行符。

<textarea cols="30" rows="10"></textarea>

其额外的属性为：

  • cols=20，可见宽度
  • rows，控制可见文本的行数
  • wrap=soft，表示控件是如何包装文本的。可能的值：hard 或 soft

注意设置默认值的不同：

  • <textarea> default value </textarea>，textarea本质上是个文本元素。
  • <input value="default value">，input是一个空元素。

2. 下拉内容

  • 可以在option上使用selected属性来指定其为默认值；
  • 可以使用optgroup来实现选项的分组，可以设置label属性为每一组设置组名。
  • 如果设置了value，则选择该项时发送的是value，否则发送option中的值。
  • 可以设置multiple属性，使得选择框成为一个多选框。（此时其不会显示为一个下拉菜单，而多行显示）

单选框示例：

<select id="groups" name="groups">
  <optgroup label="fruits">
    <option>Banana</option>
    <option selected>Cherry</option>
    <option>Lemon</option>
  </optgroup>
  <optgroup label="vegetables">
    <option value=1>Carrot</option>
    <option>Eggplant</option>
    <option>Potato</option>
  </optgroup>
</select>

多选框示例：

<select multiple id="multi" name="multi">
  <option>Banana</option>
  <option>Cherry</option>
  <option>Lemon</option>
</select>

3. 可选中项

即单选框（radio）和复选框（checkbox）。

值得注意的是，这些小部件与其他表单小部件不一样。对于大多数表单部件，一旦表单提交，所有具有name属性的小部件都会被发送，即使没有任何值被填。对于可选中项，只有在勾选时才发送它们的值。如果他们没有被勾选，就不会发送任何东西，甚至连他们的名字也没有。

为了获得最大的可用性和可访问性，建议您在<fieldset>中包围每个相关项目的列表，并使用<legend>提供对列表的全面描述。每个单独的<label>/<input>元素都应该包含在它自己的列表项中(或者类似的)。正如在示例中显示的。

您还需要为这些类型的输入提供value属性，如果您想让它们具有意义——如果没有提供任何值，则复选框和单选按钮被赋予一个 on值。

其默认值设置通过设置属性checked

  1. 复选框

    <input type="checkbox" checked id="carrots" name="carrots" value="carrots">

  2. 单选按钮

    如果多个radio的name是相同的值，则同一时间只能有一个被选中。

    <fieldset>
<legend>what is your favorite meal?</legend>
<ul>
    <li>
    <label for="soup">soup</label>
    <input type="radio" checked id="soup" name="meal" value="soup">
    </li>
    <li>
    <label for="curry">curry</label>
    <input type="radio" id="curry" name="meal" value="curry">
    </li>
    <li>
    <label for="pizza">pizza</label>
    <input type="radio" id="pizza" name="meal" value="pizza">
    </li>
</ul>
</fieldset>
what is your favorite meal?

4. 按钮

有两种方式来创建按钮：input和button。两者有以下的不同：

  • <button>元素允许您在它们的标签中使用HTML内容，这些内容被插入到打开和关闭<button> 标签中。另一方面，<input>元素是空元素;它们的标签插入在value属性中，因此只接受纯文本内容。
  • 使用<button>元素，可以有一个不同于按钮标签的值(通过设置value中的属性值)。

创建的按钮分为3类：

  1. submit

    <button type="submit">
    This a <br><strong>submit button</strong>
</button>

<input type="submit" value="This is a submit button">

    将表单数据发送到服务器。对于<button> 元素, 省略 type 属性 (或是一个无效的 type 值) 的结果就是一个提交按钮.

  2. reset

    <button type="reset">
    This a <br><strong>reset button</strong>
</button>

<input type="reset" value="This is a reset button">

    将所有表单小部件重新设置为它们的默认值。

  3. anonymous

    <button type="button">
    This an <br><strong>anonymous button</strong>
</button>

<input type="button" value="This is an anonymous button">

    没有自动生效的按钮，但是可以使用JavaScript代码进行定制。

5. 高级表单控件

  1. 数字

    <input type="number" name="age" id="age" min="1" max="10" step="2">

  2. 滑块

    <input type="range" name="beans" id="beans" min="0" max="500" step="10">

    滑块的一个问题是，它们不提供任何形式的视觉反馈，以了解当前的值是什么。您需要使用JavaScript来添加这一点，但这相对来说比较容易。

  3. 日期时间选择器

    <input type="datetime-local" name="datetime" id="datetime">

    <input type="month" name="month" id="month">

    <input type="time" name="time" id="time">

    <input type="week" name="week" id="week">

    所有日期和时间控制都可以使用min和max属性来约束。

    <label for="myDate">When are you available this summer?</label>
<input type="date" name="myDate" min="2013-06-01" max="2013-08-31" id="myDate">

  4. 拾色器

    <input type="color" name="color" id="color">

6. 其他小部件

  1. 文件选择器

    <input type="file" name="file" id="file" accept="image/*" multiple>

  2. 隐藏属性

    有时候，由于为了方便技术原因，有些数据是用表单发送的，但不显示给用户。要做到这一点，您可以在表单中添加一个不可见的元素。要做到这一点，需要使用<input>将它的type属性设置为hidden值。

    <input type="hidden" id="timestamp" name="timestamp" value="1286705410">

  3. 图像按钮，其使用<input type="image">开头，其他的属性和img元素相同，其在HTML上展现为一张图片。当点击这张图片时，将会发送在图片上的xy坐标（其实发送的是一个如此的url：http://foo.com?pos.x=123&pos.y=456）

    <input type="image" alt="Click me!" src="my-img.png" width="80" height="30" />

  4. 进度条（不再是input）

    <progress max="100" value="75">75/100</progress>

    75/100

  5. 仪表盘（不再是input）

    <meter min="0" max="100" value="75" low="33" high="66" optimum="50">75</meter>

    75

这些表单控件没必要死记硬背，知道有这么个东西，当用到的时候再回来查就行了。

提交表单后

web基于非常基本的客户端/服务器体系结构，可以总结如下:客户端(通常是web浏览器)向服务器发送请求(大多数情况下是Apache、Nginx、IIS、Tomcat等web服务器)，使用HTTP 协议。服务器使用相同的协议来回答请求。

在客户端，HTML表单只不过是一种方便的用户友好的方式，可以配置HTTP请求将数据发送到服务器。这使用户能够提供在HTTP请求中传递的信息。

客户端：如果发送数据

<form>元素定义了如何发送数据。它的所有属性都是为了让您配置当用户点击提交按钮时发送的请求。两个最重要的属性是action和method。

  1. action，其定义了要发送的位置，可以是绝对的URL（如http://foo.com），也可以是相对的URL（如/somwhere_else），也可以不填，则表单被发送到表单出现的相同页面。

    注意，尽量使用https协议，其将要发送的数据进行加密后发送，这样会比较安全。

  2. method，其定义了发送数据的方式，其中最常见的是get和post方法。

HTTP请求

由两部分组成：一个包含关于浏览器功能的全局元数据集的头部，和一个包含服务器处理特定请求所需信息的主体。

  • get方法，是浏览器使用的方法，请求服务器返回给定的资源:“嘿，服务器，我想要得到这个资源。”在这种情况下，浏览器发送一个空的主体。由于主体是空的，如果使用该方法发送一个表单，那么发送到服务器的数据将被追加到URL。

    比如下面的表单：

    <form action="http://foo.com" method="get">
<div>
    <label for="say">What greeting do you want to say?</label>
    <input name="say" id="say" value="Hi">
</div>
<div>
    <label for="to">Who do you want to say it to?</label>
    <input name="to" id="to" value="Mom">
</div>
<div>
    <button>Send my greetings</button>
</div>
</form>

    会发送www.foo.com/?say=Hi&to=Mom，所以我们得到的HTTP请求为：

    GET /?say=Hi&to=Mom HTTP/2.0
Host: foo.com

  • post方法，这是浏览器在询问响应时使用与服务器通信的方法，该响应考虑了HTTP请求正文中提供的数据:“嘿，服务器，看一下这些数据，然后给我回一个适当的结果。”如果使用该方法发送表单，则将数据追加到HTTP请求的主体中。

我们只是将上面那一段代码的method="get"修改为method="post"，将会得到下面的数据：

POST / HTTP/2.0
Host: foo.com
Content-Type: application/x-www-form-urlencoded
Content-Length: 13

say=Hi&to=Mom

查看HTTP请求可以通过Chrome Developer Tools，比如在下面：


至于两个方法什么时候使用：

  • 如果您需要发送一个密码(或其他敏感数据)，永远不要使用GET方法否则数据会在URL栏中显示，这将非常不安全。
  • 如果您需要发送大量的数据，那么POST方法是首选的，因为一些浏览器限制了URL的大小。此外，许多服务器限制它们接受的URL的长度。

在服务器端：检索数据

无论选择哪种HTTP方法，服务器都会接收一个字符串并解析，以便将数据作为键/值对序列获取。您访问这个序列的方式取决于您使用的开发平台以及您可能使用的任何特定框架。您使用的技术也决定了如何处理密钥副本；通常，最近收到的密钥的值是优先的。

以下是python的例子，原始教程中还提供了PHP的例子，但我就不用了：

# 这里使用flask 框架完成
from flask import Flask, render_template, request
app = Flask(__name__)

@app.route('/', methods=['GET', 'POST'])
def form():
    return render_template('form.html')

@app.route('/hello', methods=['GET', 'POST'])
def hello():
    return render_template('greeting.html', say=request.form['say'], to=request.form['to'])

if __name__ == "__main__":
    app.run()

特殊案例：发送文件

用HTML表单发送文件是一个特殊的例子。文件是二进制数据——或者被认为是这样的——而所有其他数据都是文本数据。由于HTTP是一种文本协议，所以处理二进制数据有特殊的要求。

这时候需要使用enctype属性，这个属性指代的是请求头中content-type的值，正常是application/x-www-form-urlencoded，即“这是已编码为URL参数的表单数据”。但文件不能放在URL中，所以需要另外设置。

<!-- 1. 永远用post -->
<!-- 2. 将enctype改为如下的值 -->
<!-- 3. 使用<input type="file"> -->
<form method="post" enctype="multipart/form-data">
  <div>
    <label for="file">Choose a file</label>
    <input type="file" id="file" name="myFile">
  </div>
  <div>
    <button>Send the file</button>
  </div>
</form>
